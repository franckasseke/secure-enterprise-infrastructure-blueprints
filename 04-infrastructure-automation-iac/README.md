# 04 - Infrastructure Automation and IaC

This domain covers Ansible, Terraform, and policy checks used to make infrastructure deployment repeatable and auditable.

## V1 project targets

1. Ansible Linux Hardening Role
2. Terraform AWS Secure VPC Lab

## Later project areas

- Patch management automation
- Terraform Azure baseline
- IaC scanning with Checkov or tfsec
- Policy-as-Code practices

## Purpose

Automation is treated as a security and compliance accelerator, not only as a productivity tool.
