# Lab Methodology

Each project follows a delivery-oriented method.

## Standard workflow

1. Scope the objective.
2. Describe the environment.
3. Identify risks and controls.
4. Build the solution.
5. Break or test the weak points.
6. Fix and improve.
7. Validate with evidence.
8. Document the result.
9. Clean up resources when needed.

## Expected evidence

A project should include proof such as command outputs, screenshots, logs, reports, scan results, or configuration snippets.

## Publication rule

A project is not considered ready for publication until it has a clear objective, implementation notes, validation evidence, and a conclusion.
