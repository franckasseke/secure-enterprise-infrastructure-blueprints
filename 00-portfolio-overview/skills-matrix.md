# Skills Matrix

This matrix maps each portfolio domain to the skills demonstrated and the type of proof expected.

## Maturity levels

| Level | Meaning |
|---|---|
| Foundation | Concepts understood and practiced in small labs. |
| Operational | Can implement, troubleshoot, and validate in a realistic environment. |
| Advanced in progress | Can design patterns, automate, compare options, and document trade-offs. |

## Portfolio skill map

| Domain | Skills | Current proof target |
|---|---|---|
| Linux Security and Compliance | Hardening, SSH, users, sudo, PAM, audit, logging, patching, secure services | OpenSCAP, SSH hardening, rsyslog, Lynis, repository integrity |
| Cloud Security and Compliance | IAM/RBAC, network security, secrets, policy, logging, governance | Azure governance, AWS VPC security, GCP IAM, multicloud control mapping |
| Cloud Native and Container Security | Docker, Kubernetes RBAC, NetworkPolicy, image scanning, runtime security, GitOps | Secure Dockerfile, image scanning, Kubernetes security labs |
| Infrastructure Automation and IaC | Ansible, Terraform, Policy-as-Code, repeatability, idempotence | Ansible hardening role, Terraform secure VPC, Checkov/tfsec scanning |
| AI Security and MLOps | AI threat modeling, secure AI use cases, risk register, local/private AI patterns | AI risk register, secure private LLM platform, secure AI PoC boundaries |

## Rule for publishing skills

A skill should be displayed prominently only when the repository contains at least one documented project proving it.
