# Project Documentation Template

Use this template for each published project.

## Compact mode

Small labs use one `README.md` with the sections below.

## Full mode

Large projects may split the same sections into separate files.

## Sections

### 1. Context

Explain the enterprise-style problem addressed by the project.

### 2. Objectives

List the technical and security objectives.

### 3. Architecture and Environment

Describe machines, networks, services, tools, and assumptions.

### 4. Security Controls

Map the project to security controls, standards, or good practices when relevant.

### 5. Implementation

Document the configuration and commands used.

### 6. Validation Evidence

Show proof that the project works as expected.

### 7. Troubleshooting

Document errors, diagnosis, and corrections.

### 8. Cleanup

Explain how to remove resources, reset changes, or avoid unnecessary cost.

### 9. References

List official documentation, benchmarks, or standards used.

## Minimum rule

Every published project must include validation evidence.
