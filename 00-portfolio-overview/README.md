# Portfolio Overview

This section explains the structure, positioning, methodology, and documentation rules of the portfolio.

The portfolio is built to demonstrate practical capability in enterprise infrastructure security through documented projects, validation evidence, and professional delivery notes.

## Core documents

- `career-positioning.md`: professional positioning and target roles.
- `skills-matrix.md`: technical domains, current maturity, and proof points.
- `architecture-map.md`: logical structure of the portfolio domains.
- `lab-methodology.md`: working method used for each project.
- `TEMPLATE.md`: reusable project documentation template.

## Core portfolio domains

1. Linux Security and Compliance
2. Cloud Security and Compliance
3. Cloud Native and Container Security
4. Infrastructure Automation and IaC
5. AI Security and MLOps

## Rule

A skill should be highlighted only when there is a corresponding documented proof point.
