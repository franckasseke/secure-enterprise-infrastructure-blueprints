# Architecture — Secure Local LLM Operations with Ollama

## Objective

This document describes the technical architecture of the local LLM workstation used in this lab.

The objective is to run an AI assistant locally on a Linux system while reducing exposure of sensitive operational information.

---

## Logical architecture

```text
User / Consultant
      |
      | CLI prompt / local API request
      v
+-----------------------------+
| Linux Workstation or VM     |
|                             |
|  Ollama runtime             |
|  Local API: 127.0.0.1:11434 |
|                             |
|  Local model cache          |
|  - deepseek-r1:7b           |
|  - qwen2.5:7b              |
|  - qwen2.5-coder:7b        |
|  - llama3.2:3b             |
+-----------------------------+
```

---

## Main components

### Linux host

The Linux host provides:

- the operating system;
- the shell environment;
- systemd service management;
- firewall controls;
- local storage for model files;
- optional GPU acceleration depending on hardware.

### Ollama runtime

Ollama provides:

- local model execution;
- model download and management;
- a command-line interface;
- a local HTTP API;
- local model storage.

### DeepSeek-R1 model

DeepSeek-R1 is used as the experimental reasoning model for:

- Linux troubleshooting;
- command explanation;
- script review;
- documentation drafting;
- structured incident analysis.

---

## Data flow

```text
1. The consultant writes a prompt locally.
2. Ollama sends the prompt to the local model runtime.
3. The model generates an answer locally.
4. The response is returned to the consultant.
5. No external AI service is required for inference after the model is downloaded.
```

Important limitation:

```text
The model does not automatically know recent external information unless it was included in its training data or provided in the prompt.
```

---

## Network exposure model

Default target state:

```text
Ollama API reachable only from localhost.
```

Validation:

```bash
ss -lntp | grep 11434
curl http://127.0.0.1:11434/api/tags
```

If Ollama is reachable from another machine, this must be treated as an exposure risk unless explicitly required and protected.

---

## Security assumptions

This lab assumes:

- the host is controlled by the consultant;
- the downloaded models are trusted enough for experimental local use;
- no production secrets are pasted into prompts;
- the workstation is not intentionally exposed to the public Internet;
- outputs generated by the model must be verified before operational use.

---

## Recommended production evolution

For a real enterprise-grade platform, this local lab should evolve toward:

- isolated VM or container runtime;
- VPN-only access;
- TLS reverse proxy;
- authentication and authorization;
- centralized logs;
- monitoring;
- encrypted storage;
- model governance;
- documented prompt handling policy.
